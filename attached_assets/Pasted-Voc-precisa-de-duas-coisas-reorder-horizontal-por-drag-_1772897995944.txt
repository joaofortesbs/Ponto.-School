Você precisa de duas coisas: reorder horizontal por drag‑and‑drop, e transição suave quando muda a aba; isso é 100% controlável só com estado de abas + lógica de drag + animações de posição.

1. Conceito simples (visão de produto)
O usuário segura em cima da aba (Home, Atividades…) e arrasta para a esquerda ou direita.

Enquanto ele arrasta, as outras abas “dão espaço”, reposicionando visualmente de forma suave para indicar onde a aba vai cair.

Ao soltar, a ordem das abas é atualizada, e essa nova ordem passa a ser a padrão daquele usuário.

Independentemente da posição, clicar em qualquer aba faz a navegação suave: o conteúdo dentro do card troca com uma animação (fade/slide) sem travar o resto da interface.

2. Visão técnica para o dev (sem código literal)
2.1. Estrutura de dados das abas
Mantenha um array ordenado de abas no estado, por exemplo:

Cada item com: id, sessionId, label, isActive, width, xPosition.

xPosition é a posição horizontal alvo da guia dentro do topo do card (usado tanto para desenhar o path quanto para posicionar o label).

A ordem do array define a ordem visual (da esquerda para direita); reordenar = reordenar esse array.

2.2. Drag‑and‑drop de abas
Independente do framework, a lógica é:

Início do drag

Ao pressionar em uma aba (mouse down / touch start) reconhecida como “área de drag”:

Guarde draggedTabId.

Guarde startX (posição inicial do cursor) e startOrderIndex (índice da aba no array).

Defina um estado isDragging para essa aba.

Durante o drag (move)

Calcule deltaX = currentPointerX - startX.

A posição visual da aba arrastada vira xPosition = originalX + deltaX (ela acompanha o dedo/mouse).

Para cada outra aba, calcule se a aba arrastada cruzou o centro dela:

Se cruzou para a direita, você considera que a aba arrastada deve ir para depois daquela; se cruzou para a esquerda, antes.

Sempre que o índice alvo muda, atualize o array ordenado e recalibre os xPosition de todas as abas (as outras “andam” para abrir espaço).

Faça esse reposicionamento com transição CSS (transform: translateX + transition) para ficar suave.

Fim do drag (drop)

No mouse up / touch end:

Fixe o índice final da aba no array.

Zere o deltaX visual; a aba passa a usar seu xPosition calculado pela nova ordem.

Limpe draggedTabId e isDragging.

Opcional: persista a nova ordem no backend para restaurar na próxima sessão.

Você pode implementar isso com API nativa de eventos ou usar uma lib de drag and drop horizontal (por exemplo, inspirações em componentes de tabs arrastáveis em React/Angular).

2.3. Integração com a borda‑aba (path)
Para manter o desenho da borda correto enquanto arrasta:

A cada frame relevante (quando xPosition das abas muda), recalcule:

As faixas horizontais de cada aba (tabStartX, tabEndX), normalmente xPosition ± metade da largura da aba.

A partir dessas faixas, gere o path do topo exatamente como você já está fazendo (subidas da borda para formar a aba, vales entre elas).

Enquanto arrasta, você pode:

Atualizar o path em tempo real (efeito mais rico), ou

Atualizar só quando o índice alvo muda (menos cálculos, mas ainda visualmente suave).

2.4. Navegação suave entre conteúdos
A aba ativa é controlada por um activeTabId.

Ao clicar em uma aba (mesmo arrastável), você só muda activeTabId; o componente de conteúdo recebe essa mudança e faz a transição.

Recomendações para transição:

Use uma animação simples:

opacity (fade) e/ou

transform: translateX leve (10–20 px) para dar sensação de “troca de painel”.
​

Garanta que a animação é rápida (150–250 ms) para não atrasar o fluxo.

Não mexa na posição das abas durante a troca; isso evita sensação de instabilidade.

2.5. Acessibilidade e UX
Além de drag, mantenha:

Clique simples para ativar aba.

Teclas de seta esquerda/direita para navegar entre abas, com role="tablist" e role="tab" para screen readers.
​

O estado de drag deve ser opcional; se o usuário só clicar, nada de arrastar deve acontecer (evite “arrastos” acidentais em cliques curtos).

Resumo para o dev:

Modelar abas com posições explícitas.

Implementar drag horizontal que altera índice no array ao cruzar centros das outras abas.

Recalcular path da borda a partir dessa nova ordem.

Animações: transitions em transform para as guias, e uma animação simples no painel de conteúdo ao trocar activeTabId.
