Objetivo
Eliminar o problema de "água contaminada no contexto" — corrigir a raiz do porquê resultados irrelevantes chegam ao LLM de gerar_conteudo_atividades, implementar um pipeline de 2 estágios de qualidade (como Perplexity), e garantir que o contexto injetado seja denso, relevante e verificado antes de chegar ao LLM de geração.

Diagnóstico — "A Água Contaminada" Explicada em Detalhe
O que acontece hoje (o problema real, com números)
O scorer atual usa esta fórmula:

rawScore = (domainScore × 0.35) + (semanticScore × 0.30) + (eduBoost × 0.20) + boosts educacionais (até 0.38)
finalScore = rawScore × typeMultiplier (1.25 para academic)
O problema crítico: `domainScore × 0.35 + eduBoost × 0.20 = 55% do score é autoridade de domínio ANTES de relevância semântica.**

Simulação real — SciELO publica artigo sobre alimentos funcionais, busca é sobre Revolução Francesa:

SciELO "Os alimentos funcionais: a nova fronteira da indústria alimentar":
  domainScore = 0.90 × 0.35   = 0.315  (SciELO é sempre 0.90)
  eduBoost    = 0.35 × 0.20   = 0.070  (tier 'alta' = eduBoost 0.35)
  semanticScore = 0  × 0.30   = 0.000  (zero termos de "revolução francesa")
  typeMultiplier = 1.25 (source_type = academic)
  ──────────────────────────────────────
  finalScore = (0.315 + 0.070) × 1.25 = 0.481
  + pedagogicoBoost (se snippet contém "aprendizagem") = +0.05
  + brasilBoost = +0.08
  ──────────────────────────────────────
  finalScore = 0.621  ← resultado irrelevante no TOP 3
Este artigo entra no prompt_context com 800 chars de conteúdo sobre alimentos via Jina Reader. O LLM de gerar_conteudo recebe esse conteúdo e ou o usa erroneamente, ou o ignora — mas ocupa espaço de contexto valioso.

O que a ciência diz sobre contexto contaminado
Research recente (2024-2025) confirma que documentos irrelevantes no contexto RAG causam degradação ativa — não são neutros, são "hard negatives" que ativamente enganam o modelo:

Inserir apenas 25% de documentos irrelevantes pode reduzir a acurácia do LLM em até 30%
"Hard negatives" (documentos parecidos mas errados) são piores que ruído aleatório
Aumentar k (número de resultados) sem filtro de qualidade: performance sobe até k=10 e depois declina
Modelos com contexto longo sofrem mais — o LLM "processa" o texto irrelevante e se confunde
Como os líderes de mercado resolvem
Perplexity — Pipeline de 2 estágios:

Stage 1: Retrieval de alta cobertura (30-100 candidatos), sem filtro de qualidade ainda
Stage 2: Cross-encoder reranker (modelo que analisa QUERY + DOCUMENTO juntos) — aplica threshold duro: qualquer resultado abaixo do corte é descartado, não penalizado. Chamam de "sculpting"
Sub-document scoring: extrai o trecho mais relevante dentro do documento, não o documento inteiro
Context compression: trunca para o orçamento de tokens priorizando relevância, não posição
Manus AI — Context Engineering:

Após cada ferramenta executada, valida se o resultado é útil para o objetivo atual
Resultados validados ficam no contexto; resultados inúteis vão para "arquivo compactado"
Só carrega no LLM o que passou pela validação
Princípio: "o LLM só deve ver o que é relevante para a próxima ação"
Genspark — Fact-checking cruzado:

Múltiplos modelos analisam os resultados de forma independente
Consenso entre modelos determina o que vai para o contexto final
Resultados sem consenso são descartados
O que o sistema atual tem que os concorrentes NÃO têm
39 domínios brasileiros curados com autoridade real
BNCC integration nativa
Pipeline completo (pesquisa → atividade → calendário)
Custo ~$0.001/busca vs. Manus ($10+/sessão) e Genspark
O gap é só de qualidade de filtragem — a estrutura de pesquisa já é competitiva.

function validateContentRelevance(content, queryTerms) {
  if (!content || queryTerms.length === 0) return { relevant: false, score: 0 };
  const contentLower = content.toLowerCase();
  const matchCount = queryTerms.filter(term => contentLower.includes(term)).length;
  const relevanceRatio = matchCount / queryTerms.length;
  return {
    relevant: matchCount >= 1,               // ao menos 1 termo do query
    score: relevanceRatio,                    // 0.0 a 1.0
    matchCount,
    label: matchCount === 0 ? 'invalido'
         : relevanceRatio < 0.3 ? 'baixa'
         : relevanceRatio < 0.6 ? 'media'
         : 'alta'
  };
}

Detalhes precisos:

Nova função em query-planner.js — Pointwise LLM Reranking (abordagem mais rápida, O(1) por batch):

export async function rerankWithLLM(query, results, options = {}) {
  const { groqApiKey, topN = 8, timeoutMs = 5000 } = options;
  if (!groqApiKey || !results || results.length === 0) return results;
  
  const candidates = results.slice(0, 15); // pega top 15 do scoring inicial
  
  const candidatesList = candidates.map((r, i) =>
    `[${i}] "${r.title.slice(0, 80)}" — ${r.source_label || r.provider || 'Web'}\n` +
    `    ${(r.snippet || '').slice(0, 120)}`
  ).join('\n\n');
  
  const prompt = `Você é especialista em educação brasileira K-12.
Query do professor: "${query}"
Avalie a RELEVÂNCIA EDUCACIONAL de cada resultado para esta query específica.
Responda APENAS com JSON válido, sem markdown:
{"scores":[{"idx":0,"score":8},{"idx":1,"score":2},...]}`
Critérios de score (0-10):
- 10: Diretamente sobre o tema, para professores, contexto BR
- 7-9: Relacionado ao tema, educacional, útil
- 4-6: Parcialmente relevante ou periférico
- 1-3: Pouco relacionado
- 0: Completamente irrelevante para "${query}"
Resultados para avaliar:
${candidatesList}`;
  
  try {
    const controller = new AbortController();
    const timeoutId = setTimeout(() => controller.abort(), timeoutMs);
    
    const response = await fetch('https://api.groq.com/openai/v1/chat/completions', {
      method: 'POST',
      signal: controller.signal,
      headers: {
        'Authorization': `Bearer ${groqApiKey}`,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        model: 'llama-3.1-8b-instant',
        messages: [{ role: 'user', content: prompt }],
        temperature: 0.0,
        max_tokens: 400,
      }),
    });
    
    clearTimeout(timeoutId);
    
    if (!response.ok) return results; // falha silenciosa
    
    const data = await response.json();
    const rawContent = data?.choices?.[0]?.message?.content || '';
    const parsed = JSON.parse(rawContent.replace(/```json\n?|\n?```/g, '').trim());
    
    if (!parsed.scores || !Array.isArray(parsed.scores)) return results;
    
    // Criar mapa idx → groq_score
    const scoreMap = {};
    for (const s of parsed.scores) {
      if (typeof s.idx === 'number' && typeof s.score === 'number') {
        scoreMap[s.idx] = s.score;
      }
    }
    
    // Aplicar reranking:
    // - groq_score < 3: penalizar (× 0.35) — resultado irrelevante descartado na prática
    // - groq_score >= 3 e < 7: neutro
    // - groq_score >= 7: boost (× 1.15)
    const reranked = candidates.map((r, i) => {
      const gs = scoreMap[i] ?? 5; // default neutro se não retornou
      const multiplier = gs < 3 ? 0.35 : gs >= 7 ? 1.15 : 1.0;
      return {
        ...r,
        score: Math.min(1.0, r.score * multiplier),
        groq_relevance_score: gs,
      };
    });

O que acontece hoje (o problema real, com números)
O scorer atual usa esta fórmula:

rawScore = (domainScore × 0.35) + (semanticScore × 0.30) + (eduBoost × 0.20) + boosts educacionais (até 0.38)

O que a ciência diz sobre contexto contaminado
Research recente (2024-2025) confirma que documentos irrelevantes no contexto RAG causam degradação ativa — não são neutros, são "hard negatives" que ativamente enganam o modelo:

Inserir apenas 25% de documentos irrelevantes pode reduzir a acurácia do LLM em até 30%
"Hard negatives" (documentos parecidos mas errados) são piores que ruído aleatório
Aumentar k (número de resultados) sem filtro de qualidade: performance sobe até k=10 e depois declina
Modelos com contexto longo sofrem mais — o LLM "processa" o texto irrelevante e se confunde
Como os líderes de mercado resolvem
Perplexity — Pipeline de 2 estágios:

Stage 1: Retrieval de alta cobertura (30-100 candidatos), sem filtro de qualidade ainda
Stage 2: Cross-encoder reranker (modelo que analisa QUERY + DOCUMENTO juntos) — aplica threshold duro: qualquer resultado abaixo do corte é descartado, não penalizado. Chamam de "sculpting"
Sub-document scoring: extrai o trecho mais relevante dentro do documento, não o documento inteiro
Context compression: trunca para o orçamento de tokens priorizando relevância, não posição
Manus AI — Context Engineering:

Após cada ferramenta executada, valida se o resultado é útil para o objetivo atual
Resultados validados ficam no contexto; resultados inúteis vão para "arquivo compactado"
Só carrega no LLM o que passou pela validação
Princípio: "o LLM só deve ver o que é relevante para a próxima ação"
Genspark — Fact-checking cruzado:

Múltiplos modelos analisam os resultados de forma independente
Consenso entre modelos determina o que vai para o contexto final
Resultados sem consenso são descartados
O que o sistema atual tem que os concorrentes NÃO têm

BNCC integration nativa

Custo ~$0.001/busca vs. Manus ($10+/sessão) e Genspark

function validateContentRelevance(content, queryTerms) {
  if (!content || queryTerms.length === 0) return { relevant: false, score: 0 };
  const contentLower = content.toLowerCase();
  const matchCount = queryTerms.filter(term => contentLower.includes(term)).length;
  const relevanceRatio = matchCount / queryTerms.length;
  return {
    relevant: matchCount >= 1,               // ao menos 1 termo do query
    score: relevanceRatio,                    // 0.0 a 1.0
    matchCount,
    label: matchCount === 0 ? 'invalido'
         : relevanceRatio < 0.3 ? 'baixa'
         : relevanceRatio < 0.6 ? 'media'
         : 'alta'
  };
}

export async function rerankWithLLM(query, results, options = {}) {
  const { groqApiKey, topN = 8, timeoutMs = 5000 } = options;
  if (!groqApiKey || !results || results.length === 0) return results;
  
  const candidates = results.slice(0, 15); // pega top 15 do scoring inicial
  
  const candidatesList = candidates.map((r, i) =>
    `[${i}] "${r.title.slice(0, 80)}" — ${r.source_label || r.provider || 'Web'}\n` +
    `    ${(r.snippet || '').slice(0, 120)}`
  ).join('\n\n');
  
  const prompt = `Você é especialista em educação brasileira K-12.
Query do professor: "${query}"
Avalie a RELEVÂNCIA EDUCACIONAL de cada resultado para esta query específica.
Responda APENAS com JSON válido, sem markdown:
{"scores":[{"idx":0,"score":8},{"idx":1,"score":2},...]}`
Critérios de score (0-10):
- 10: Diretamente sobre o tema, para professores, contexto BR
- 7-9: Relacionado ao tema, educacional, útil
- 4-6: Parcialmente relevante ou periférico
- 1-3: Pouco relacionado
- 0: Completamente irrelevante para "${query}"
Resultados para avaliar:
${candidatesList}`;
  
  try {
    const controller = new AbortController();
    const timeoutId = setTimeout(() => controller.abort(), timeoutMs);
    
    const response = await fetch('https://api.groq.com/openai/v1/chat/completions', {
      method: 'POST',
      signal: controller.signal,
      headers: {
        'Authorization': `Bearer ${groqApiKey}`,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        model: 'llama-3.1-8b-instant',
        messages: [{ role: 'user', content: prompt }],
        temperature: 0.0,
        max_tokens: 400,
      }),
    });
    
    clearTimeout(timeoutId);
    
    if (!response.ok) return results; // falha silenciosa
    
    const data = await response.json();
    const rawContent = data?.choices?.[0]?.message?.content || '';
    const parsed = JSON.parse(rawContent.replace(/```json\n?|\n?```/g, '').trim());
    
    if (!parsed.scores || !Array.isArray(parsed.scores)) return results;
    
    // Criar mapa idx → groq_score
    const scoreMap = {};
    for (const s of parsed.scores) {
      if (typeof s.idx === 'number' && typeof s.score === 'number') {
        scoreMap[s.idx] = s.score;
      }
    }
    
    // Aplicar reranking:
    // - groq_score < 3: penalizar (× 0.35) — resultado irrelevante descartado na prática
    // - groq_score >= 3 e < 7: neutro
    // - groq_score >= 7: boost (× 1.15)
    const reranked = candidates.map((r, i) => {
      const gs = scoreMap[i] ?? 5; // default neutro se não retornou
      const multiplier = gs < 3 ? 0.35 : gs >= 7 ? 1.15 : 1.0;
      return {
        ...r,
        score: Math.min(1.0, r.score * multiplier),
        groq_relevance_score: gs,
      };
    });
    
    // Reordenar pelos novos scores, adicionar resultados não-candidatos no final
    const rerankedSorted = reranked.sort((a, b) => b.score - a.score);
    const remainingResults = results.slice(15); // resultados além dos 15 candidatos
    
    console.log(`[LLMReranker] Groq reclassificou ${candidates.length} resultados para query "${query.slice(0, 40)}"`);
    return [...rerankedSorted, ...remainingResults];
    
  } catch (err) {
    console.warn('[LLMReranker] Falha silenciosa, mantendo ranking original:', err.message);
    return results; // falha silenciosa sempre
  }
}
