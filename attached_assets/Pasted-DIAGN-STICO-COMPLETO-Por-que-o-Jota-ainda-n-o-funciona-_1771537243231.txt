POR QUE REGEX AINDA EXISTE E O QUE CAUSA
As fast rules (que substituíram o regex puro) existem porque o LLM do SmartRouter falha frequentemente. O real problema é circular:

O SmartRouter envia um prompt de routing para a IA
A IA recebe TUDO como user message (sem system prompt separado)
A IA fica confusa e retorna JSON malformado
O sistema cai nas fast rules
Mesmo quando as fast rules acertam, o próximo passo (handleDirectResponse) tem o MESMO problema de prompt misturado
