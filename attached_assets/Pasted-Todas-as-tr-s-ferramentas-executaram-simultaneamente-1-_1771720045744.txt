### 1.5. Fase de Planejamento de Queries (Query Planning)

**SIM, existe uma fase explícita de planejamento.**

```python
# Processo de Query Planning

class QueryPlanner:
    """
    Planejador de queries antes da execução
    """
    
    def planejar(self, mensagem_usuario, contexto):
        """
        Planeja estratégia de busca antes da execução
        """
        
        # STEP 1: Análise de Complexidade
        complexidade = self.avaliar_complexidade(mensagem_usuario)
        
        if complexidade == "SIMPLES":
            # Query única, direta
            return self.estrategia_simples()
        
        elif complexidade == "MODERADA":
            # Decomposição em 2-4 subqueries
            return self.estrategia_decomposicao()
        
        elif complexidade == "COMPLEXA":
            # Deep research com múltiplas iterações
            return self.estrategia_deep_research()
        
        # STEP 2: Identificação de Aspectos Múltiplos
        aspectos = self.identificar_aspectos(mensagem_usuario)
        # Ex: "recursos de genética" pode ter aspectos:
        # - Teóricos
        # - Práticos/experimentais
        # - Vídeos educacionais
        # - Artigos acadêmicos
        
        # STEP 3: Mapeamento Aspecto → Query
        queries = []
        for aspecto in aspectos:
            query = self.formular_query_para_aspecto(aspecto)
            queries.append(query)
        
        # STEP 4: Otimização de Queries
        queries_otimizadas = self.otimizar_queries(queries)
        # Remoção de redundâncias
        # Balanceamento de especificidade
        # Limitação a 6 queries máximo
        
        # STEP 5: Seleção de Ferramentas
        tools_map = self.mapear_ferramentas(aspectos)
        # Ex: aspecto "acadêmico" → scholar_search
        #     aspecto "visual" → video_search
        #     aspecto "geral" → batch_web_search
        
        return {
            'queries': queries_otimizadas,
            'tools': tools_map,
            'strategy': 'parallel' if self.sao_independentes(queries) else 'sequential'
        }
```

### 3.2. Reconciliação de Informações Contraditórias

```python
class ConflictResolver:
    """
    Sistema de resolução de conflitos entre fontes
    """
    
    def resolver_conflito(self, informacoes_conflitantes):
        """
        Estratégia hierárquica de resolução
        """
        
        # NÍVEL 1: Verificação Temporal
        if eh_conflito_temporal(informacoes):
            return self.priorizar_por_recencia(informacoes)
            # Ex: "Ministro da Educação é X" (2025) vs. "Ministro é Y" (2023)
            # → Retorna a informação mais recente
        
        # NÍVEL 2: Hierarquia de Autoridade
        scores_autoridade = []
        for info in informacoes:
            score = self.calcular_autoridade_fonte(info.fonte)
            scores_autoridade.append((info, score))
        
        fonte_mais_autoritativa = max(scores_autoridade, key=lambda x: x[1])
        
        if fonte_mais_autoritativa[1] > THRESHOLD_AUTORIDADE_CLARA:
            return fonte_mais_autoritativa[0]
        
        # NÍVEL 3: Análise de Consenso
        clusters = self.clusterizar_informacoes_similares(informacoes)
        maior_cluster = max(clusters, key=len)
        
        if len(maior_cluster) >= 0.6 * len(informacoes):
            # 60%+ das fontes concordam
            return self.sintetizar_consenso(maior_cluster)
        
        # NÍVEL 4: Apresentação Múltipla (conflito irreconciliável)
        return self.apresentar_multiplas_perspectivas(informacoes)
    
    def calcular_autoridade_fonte(self, fonte):
        """
        Scoring de autoridade
        """
        score = 0
        
        # Domínios governamentais
        if '.gov' in fonte.url:
            score += 100
        
        # Instituições acadêmicas
        if '.edu' in fonte.url or 'scholar' in fonte.url:
            score += 90
        
        # Organizações estabelecidas
        if fonte.dominio in ['novaescola.org.br', 'mec.gov.br', 'scielo.br']:
            score += 85
        
        # Jornalismo de referência
        if fonte.dominio in ['bbc.com', 'nytimes.com', 'folha.uol.com.br']:
            score += 70
        
        # Blogs e sites pessoais
        if 'wordpress.com' in fonte.url or 'blogspot.com' in fonte.url:
            score += 20
        
        return score
    
    def apresentar_multiplas_perspectivas(self, informacoes):
        """
        Quando não dá para resolver, apresenta todas
        """
        output = "Há divergência nas fontes sobre este tópico:\n\n"
        
        for info in informacoes:
            output += f"• Segundo {info.fonte_nome} [{info.tipo_fonte}]: {info.conteudo}\n"
            output += f"  [Fonte]({info.url})\n\n"
        
        output += "Recomenda-se consultar as fontes originais para avaliação."
        
        return output
```

```python
def reranking_resultados(resultados_iniciais, query_original, contexto_usuario):
    """
    Re-ranking após obtenção inicial
    """
    
    # FEATURES PARA RE-RANKING
    features = []
    
    for resultado in resultados_iniciais:
        
        # 1. Relevância Semântica (modelo de embedding)
        relevancia_semantica = calcular_similaridade_semantica(
            query_embedding=embed(query_original),
            doc_embedding=embed(resultado.snippet + resultado.title)
        )
        
        # 2. Autoridade do Domínio
        autoridade = domain_authority_score(resultado.url)
        
        # 3. Recência
        recencia = calcular_recencia_score(resultado.data_publicacao)
        # Decay exponencial: score = e^(-lambda * days_ago)
        
        # 4. Engagement (se disponível)
        engagement = (
            resultado.views * 0.3 +
            resultado.shares * 0.5 +
            resultado.comments * 0.2
        ) if hasattr(resultado, 'engagement_metrics') else 0
        
        # 5. Tipo de Conteúdo Preferencial
        tipo_score = {
            'pdf': 1.2,  # PDFs tendem a ser mais formais
            'edu': 1.3,  # Conteúdo educacional
            'gov': 1.4,  # Governamental
            'video': 0.9,  # Depende do contexto
            'blog': 0.7
        }.get(resultado.tipo, 1.0)
        
        # 6. Personalização Contextual
        personalizacao = 0
        if contexto_usuario.get('pais') == 'Brasil':
            if '.br' in resultado.url:
                personalizacao += 0.3  # Boost para conteúdo brasileiro
        
        if contexto_usuario.get('ocupacao') == 'professor':
            if any(kw in resultado.url for kw in ['novaescola', 'educacao', 'ensino']):
                personalizacao += 0.4
        
        # SCORE COMPOSTO
        score_final = (
            relevancia_semantica * 0.40 +
            autoridade * 0.25 +
            recencia * 0.15 +
            tipo_score * 0.10 +
            personalizacao * 0.10
        )
        
        features.append((resultado, score_final))
    
    # RE-ORDENAÇÃO
    resultados_rerankeados = sorted(features, key=lambda x: x[1], reverse=True)
    
    return [r[0] for r in resultados_rerankeados]
```

```python
# Token Budget Management

LIMITS = {
    'total_context_window': 200_000,  # tokens (conforme especificado)
    'reserved_for_system': 10_000,    # prompts do sistema
    'reserved_for_user_msg': 5_000,   # mensagem do usuário

    'available_for_search': 181_000   # disponível para resultados de busca

def gerenciar_token_overflow(resultados, budget_disponivel):
    """
    Estratégias quando resultados excedem budget
    """
    
    # ESTRATÉGIA 1: Truncamento Inteligente
    if total_tokens(resultados) > budget_disponivel:
        
        # Priorizar resultados por relevância
        resultados_ordenados = ordenar_por_relevancia(resultados)
        
        # Alocação proporcional
        resultados_incluidos = []
        tokens_usados = 0
        
        for resultado in resultados_ordenados:
            tokens_resultado = contar_tokens(resultado)
            
            if tokens_usados + tokens_resultado <= budget_disponivel:
                resultados_incluidos.append(resultado)
                tokens_usados += tokens_resultado
            else:
                # Tenta truncar o snippet
                resultado_truncado = truncar_snippet(
                    resultado,
                    tokens_disponiveis=budget_disponivel - tokens_usados
                )
                if resultado_truncado:
                    resultados_incluidos.append(resultado_truncado)
                break
        
        return resultados_incluidos
    
    # ESTRATÉGIA 2: Sumarização de Clusters
    if modo == 'deep_research':
        # Para pesquisas profundas com muitos resultados
        clusters = clusterizar_resultados_por_topico(resultados)
        
        sumarios_cluster = []
        for cluster in clusters:
            sumario = sumarizar_cluster(cluster)
            sumarios_cluster.append(sumario)
        
        return sumarios_cluster
    
    # ESTRATÉGIA 3: Busca Iterativa
    if modo == 'iterative':
        # Processar em batches
        # Primeira rodada: top results
        # Rodadas subsequentes: baseadas em gaps identificados
        return processar_em_batches(resultados, budget_disponivel)
```

### 4.3. Critérios de Parada e Prevenção de Loops

```python
class DeepResearchController:
    """
    Controlador de pesquisa profunda com safeguards
    """
    
    def __init__(self):
        self.max_iterations = 6
        self.max_queries_total = 60
        self.min_new_info_threshold = 0.15  # 15% de informação nova mínima
        self.max_time_minutes = 15
        
    def deve_parar(self, estado_atual):
        """
        Critérios de parada multi-dimensional
        """
        
        # CRITÉRIO 1: Limite de Iterações
        if estado_atual.iteracao >= self.max_iterations:
            return True, "Limite de iterações atingido"
        
        # CRITÉRIO 2: Limite de Queries Totais
        if estado_atual.queries_executadas >= self.max_queries_total:
            return True, "Limite de queries atingido"
        
        # CRITÉRIO 3: Limite de Tempo
        if estado_atual.tempo_decorrido > self.max_time_minutes * 60:
            return True, "Limite de tempo atingido"
        
        # CRITÉRIO 4: Convergência de Informação
        if self.detectar_convergencia(estado_atual):
            return True, "Informação convergiu (poucos novos insights)"
        
        # CRITÉRIO 5: Completude Satisfatória
        score_completude = self.avaliar_completude(estado_atual)
        if score_completude > 0.85:  # 85% de completude
            return True, "Completude satisfatória alcançada"
        
        # CRITÉRIO 6: Retornos Decrescentes
        if self.detectar_retornos_decrescentes(estado_atual):
            return True, "Retornos decrescentes detectados"
        
        return False, "Continuar pesquisa"
    
    def detectar_convergencia(self, estado):
        """
        Detecta quando novas buscas não trazem informação nova
        """
        if len(estado.historico_iteracoes) < 2:
            return False
        
        ultima_iteracao = estado.historico_iteracoes[-1]
        penultima_iteracao = estado.historico_iteracoes[-2]
        
        # Calcular overlap de informação
        info_nova = calcular_informacao_nova(
            ultima_iteracao.resultados,
            penultima_iteracao.resultados
        )
        
        # Se menos de 15% é informação nova, convergiu
        return info_nova < self.min_new_info_threshold
    
    def avaliar_completude(self, estado):
        """
        Score de quão completa está a pesquisa
        """
        score = 0.0
        
        # Dimensão 1: Cobertura de aspectos
        aspectos_cobertos = len(estado.aspectos_explorados)
        aspectos_totais = len(estado.aspectos_identificados_inicial)
        score += (aspectos_cobertos / aspectos_totais) * 0.3
        
        # Dimensão 2: Qualidade de fontes
        tem_fontes_autoritativas = any(
            fonte.autoridade > 80 
            for fonte in estado.fontes_citadas
        )
        score += 0.2 if tem_fontes_autoritativas else 0.0
        
        # Dimensão 3: Diversidade de fontes
        diversidade = len(set(f.dominio for f in estado.fontes_citadas))
        score += min(diversidade / 10, 0.2)  # Max 0.2
        
        # Dimensão 4: Gaps remanescentes
        gaps_resolvidos = (
            estado.gaps_iniciais - len(estado.gaps_atuais)
        ) / estado.gaps_iniciais
        score += gaps_resolvidos * 0.3
        
        return score
    
    def detectar_retornos_decrescentes(self, estado):
        """
        Detecta quando o custo marginal (queries) supera o benefício marginal (info)
        """
        if len(estado.historico_iteracoes) < 3:
            return False
        
        ultimas_3 = estado.historico_iteracoes[-3:]
        
        # Queries por iteração
        queries_counts = [it.num_queries for it in ultimas_3]
        
        # Informação nova por iteração
        info_nova = [it.info_nova_percentual for it in ultimas_3]
        
        # Se queries aumentam mas info nova diminui → retornos decrescentes
        if queries_counts[-1] > queries_counts[-2] and \
           info_nova[-1] < info_nova[-2] and \
           info_nova[-1] < 0.10:  # Menos de 10% de info nova
            return True
        
        return False
```

### 4.4. Manutenção de Contexto Acumulado

```python
class DeepResearchMemory:
    """
    Sistema de memória para deep research
    """
    
    def __init__(self):
        self.knowledge_base = {
            'facts': [],           # Fatos extraídos
            'statistics': [],      # Números e dados
            'sources': [],         # Todas as fontes
            'entities': {},        # Entidades mencionadas
            'topics': {},          # Mapa de tópicos cobertos
            'gaps': [],            # Gaps identificados
            'queries_history': []  # Histórico de queries
        }
        
        self.iteration_summaries = []  # Sumário de cada iteração
        
    def adicionar_resultados(self, iteracao_num, resultados):
        """
        Processa e armazena resultados de uma iteração
        """
        
        # Extração de fatos
        novos_fatos = self.extrair_fatos(resultados)
        self.knowledge_base['facts'].extend(novos_fatos)
        
        # Deduplicação inteligente
        self.knowledge_base['facts'] = self.deduplica r_fatos(
            self.knowledge_base['facts']
        )
        
        # Extração de estatísticas
        stats = self.extrair_estatisticas(resultados)
        self.knowledge_base['statistics'].extend(stats)
        
        # Atualização de entidades
        entidades = self.extrair_entidades(resultados)
        self.atualizar_entidades(entidades)
        
        # Mapeamento de tópicos cobertos
        topicos = self.identificar_topicos(resultados)
        for topico in topicos:
            if topico not in self.knowledge_base['topics']:
                self.knowledge_base['topics'][topico] = []
            self.knowledge_base['topics'][topico].extend(
                self.filtrar_resultados_por_topico(resultados, topico)
            )
        
        # Criar sumário da iteração
        sumario = self.sumarizar_iteracao(iteracao_num, resultados)
        self.iteration_summaries.append(sumario)
        
    def identificar_novos_gaps(self):
        """
        Identifica gaps de conhecimento baseado no que já temos
        """
        gaps = []
        
        # Gap 1: Tópicos mencionados mas não explorados
        topicos_mencionados = self.extrair_topicos_mencionados()
        topicos_explorados = set(self.knowledge_base['topics'].keys())
        gaps_topicos = topicos_mencionados - topicos_explorados
        
        for topico in gaps_topicos:
            gaps.append({
                'tipo': 'topico_inexplorado',
                'descricao': f"Tópico '{topico}' mencionado mas não explorado",
                'prioridade': 'media'
            })
        
        # Gap 2: Claims sem fontes múltiplas
        facts_com_fonte_unica = [
            f for f in self.knowledge_base['facts']
            if len(f['fontes']) == 1 and f['importancia'] == 'alta'
        ]
        
        for fact in facts_com_fonte_unica:
            gaps.append({
                'tipo': 'verificacao_necessaria',
                'descricao': f"Claim '{fact['texto']}' tem apenas 1 fonte",
                'prioridade': 'alta'
            })
        
        # Gap 3: Estatísticas desatualizadas
        stats_antigas = [
            s for s in self.knowledge_base['statistics']
            if self.eh_desatualizada(s['data'])
        ]
        
        for stat in stats_antigas:
            gaps.append({
                'tipo': 'atualizacao_necessaria',
                'descricao': f"Estatística de {stat['data']} pode estar desatualizada",
                'prioridade': 'media'
            })
        
        return gaps
    
    def gerar_contexto_para_proxima_iteracao(self):
        """
        Compila contexto relevante para próxima iteração
        """
        contexto = f"""
        ═══════════════════════════════════════════════════════════
        DEEP RESEARCH CONTEXT - Iteration {len(self.iteration_summaries) + 1}
        ═══════════════════════════════════════════════════════════
        
        ## Knowledge Base Summary
        - Total Facts Collected: {len(self.knowledge_base['facts'])}
        - Statistics Gathered: {len(self.knowledge_base['statistics'])}
        - Unique Sources: {len(set(s['url'] for s in self.knowledge_base['sources']))}
        - Topics Covered: {len(self.knowledge_base['topics'])}
        - Entities Identified: {len(self.knowledge_base['entities'])}
        
        ## Previous Iterations Summary
        {self._formatar_sumarios_iteracoes()}
        
        ## Current Knowledge Gaps
        {self._formatar_gaps()}
        
        ## Recommended Next Queries
        {self._gerar_sugestoes_queries()}
        
        ═══════════════════════════════════════════════════════════
        """
        
        return contexto
```

### 4.5. Diferença Técnica: Pesquisa Rápida vs. Profunda

```
╔═══════════════════════════════════════════════════════════╗

╚═══════════════════════════════════════════════════════════╝

Arquitetura:
├─ Single-shot execution
├─ 2-6 queries paralelas
├─ Sem iteração
├─ Sem análise de gaps
└─ Resposta imediata

Processamento:
├─ Snippets apenas (não fetches conteúdo completo)
├─ Re-ranking simples
├─ Sem deduplicação avançada
└─ Síntese direta dos resultados

Tempo típico: 3-10 segundos
Queries totais: 2-6
Profundidade: Superficial a moderada

Casos de uso:
✓ Perguntas factuais simples
✓ Consultas rápidas
✓ Informação imediata necessária

───────────────────────────────────────────────────────────

╔═══════════════════════════════════════════════════════════╗
║  PESQUISA PROFUNDA (deep_research agent)                  ║
╚═══════════════════════════════════════════════════════════╝

├─ Multi-iteration loop
├─ 15-60 queries distribuídas
├─ Gap analysis entre iterações
├─ Refinamento progressivo
└─ Síntese estruturada final

Processamento:
├─ Fetching de conteúdo completo (crawler)
├─ Cross-referencing de fontes
├─ Deduplicação avançada
├─ Fact-checking
├─ Construção de knowledge base

├─ Mapa de tópicos cobertos
├─ Histórico de iterações
```
