Ótima observação! Vou te explicar o que está acontecendo com cada um dos seus pontos:

1. Card de Desenvolvimento não aparece nas operações de calendário
O "Card de Desenvolvimento" (que mostra as etapas e capacidades sendo executadas) só é criado quando o Jota segue o caminho EXECUTAR — ou seja, quando ele cria um plano de execução com etapas. Quando o calendário é acessado via CAPABILITY_DIRETA (que é o caminho correto e mais rápido), ele pula a criação do plano e executa diretamente. Por isso o card não aparece.

Solução: Criar um card de desenvolvimento simplificado para execuções diretas (CAPABILITY_DIRETA), mostrando a capability sendo executada com seu status.

2. Texto feio e mal estruturado
Ferramentas de formatação disponíveis:
O componente RichTextMessage já existe e suporta: cabeçalhos (H1-H6), parágrafos, listas, callouts (tip/warning/info/success), tabelas, código, separadores, e formatação inline (negrito, itálico, links). Todas as respostas do assistente já passam por esse componente.

Caminhos de resposta e suas formatações:
Caminho	Quando	Formatação
CONVERSAR	Conversa geral, saudações	Texto simples via handleDirectResponse — o LLM gera texto, mas sem instruções específicas de formatação rica
CAPABILITY_DIRETA	Calendário, pesquisas	Retorna resposta_para_professor — que é texto cru do mini-agente, sem formatação markdown
EXECUTAR	Criação de conteúdo	Plano completo com card de desenvolvimento + artefatos com formatação rica
O problema principal:
As respostas de calendário vêm do mini-agente de calendário como texto puro (ex: "📊 Análise de disponibilidade..."), sem usar markdown ou formatação estruturada. O RichTextMessage só formata o que recebe — se recebe texto sem markdown, mostra como texto simples. Além disso, o caminho CAPABILITY_DIRETA não usa o system prompt SYSTEM_PROMPT_CONVERSAR que instrui formatação bonita.

Como melhorar:
Adicionar formatação markdown nas respostas do calendário — Instruir o mini-agente a usar headers, listas, callouts
Pós-processar a resposta de CAPABILITY_DIRETA — Envolver o resultado bruto em um template formatado antes de enviar ao chat
Criar um card visual específico para calendário — Similar ao ArtifactCard, mas para eventos (com ícones, cores, datas formatadas)
